# コンセプト

## プロジェクト名

**熾火守（オキビモリ）** - Hookah Heat Management

## 背景と課題

シーシャ店では、複数の台を同時に管理する必要があり、各台のメンテナンス状況を適切に追跡することが重要です。以下のような課題がありました：

- 各台の最終メンテナンス内容記憶するのが困難
- 忙しい時間帯に作業の優先順位がわかりにくい
- 目的は再現性の確保ではなく、あくまで記憶のサポート
- 片手間で必要最小限の情報を残せる操作性が必要
- 複数スタッフが居る場合でも、担当する台は分かれる想定なので店内での情報共有は不要

## 解決策

各台のメンテナンス状況をリアルタイムで可視化し、ワンタップで記録できるPWAアプリケーションを提供します。

### 主な特徴

1. **即座の記録**: ワンタップでメンテナンス作業を記録
2. **視覚的な優先度**: 経過時間を色分けして表示（10分超過→黄色、15分超過→赤色）
3. **履歴管理**: 各台のメンテナンス履歴を自動保存
4. **オフライン対応**: IndexedDBによる完全なローカル動作
5. **PWA**: ホーム画面に追加してネイティブアプリのように利用可能

## ターゲットユーザー

- **シーシャ店スタッフ**: 複数台を同時管理する店舗スタッフ
- **店舗オーナー**: サービス品質を向上させたい経営者

## ユースケース

### 基本フロー

1. **セッション開始**: 客が来店し、台番号とフレーバーを記録
2. **メンテナンス記録**: すす捨て、炭交換、調整をワンタップで記録
3. **優先度確認**: 一覧画面で経過時間を確認し、次の作業を判断
4. **詳細確認**: 必要に応じて履歴や詳細情報を確認
5. **セッション終了**: 客が退店したら終了処理

### 具体的なシナリオ

**シナリオ1: 忙しい時間帯の優先順位判断**
```
状況: 4台が稼働中、2つの作業依頼が同時に発生
1. メイン画面で各台の経過時間を確認
2. 15分超過（赤色）の台を優先的に対応
3. ワンタップでメンテナンス記録
4. 次の台へ移動
```

**シナリオ2: フレーバー履歴の活用**
```
状況: 常連客が「前回と同じフレーバー」を希望
1. セッション追加画面を開く
2. フレーバー入力欄をクリック
3. 履歴から過去のフレーバーを選択
4. 素早くセッション開始
```

**シナリオ3: サービス品質の振り返り**
```
状況: 特定の台のメンテナンス頻度を確認
1. 詳細画面を開く
2. 履歴（最新10件）を確認
3. メンテナンス間隔を把握し、改善点を検討
```

## デザインフィロソフィー

### ネオンサイバーパンクテーマ

暗い店内でも見やすく、モダンで洗練された印象を与えるネオンカラーを採用。各メンテナンス作業に色を割り当て、視覚的な識別を容易にしています。

- **シアン**: すす捨て（清掃のイメージ）
- **ピンク**: 炭交換（熱のイメージ）
- **パープル**: 調整（繊細な作業のイメージ）
- **グリーン**: フレーバー名（新鮮さのイメージ）
- **イエロー/レッド**: 警告表示（緊急度のイメージ）

### UI/UX原則

1. **効率性優先**: すべての操作を最小タップ数で完了
2. **一貫性**: 同じ情報は同じ色・同じ位置に表示
3. **視認性**: 暗い環境でも見やすい高コントラスト
4. **モバイルファースト**: スマートフォンでの操作に最適化

## 技術選定の理由

### PWA (Progressive Web App)

- **インストール不要**: ブラウザから直接利用開始
- **ホーム画面追加**: ネイティブアプリのような体験
- **オフライン動作**: 完全にローカルで動作

### IndexedDB

- **プライバシー**: データがデバイス内に完結
- **高速**: サーバー通信なしで即座にアクセス
- **容量**: 十分な履歴を保存可能

### Next.js 15

- **最新技術**: App Routerによるモダンな開発体験
- **最適化**: 自動的な最適化とパフォーマンス
- **型安全**: TypeScriptによる堅牢な開発
